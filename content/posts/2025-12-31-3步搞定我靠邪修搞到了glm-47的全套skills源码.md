## 与 Claude Skills有什么区别？

眼尖的朋友已经发现了，其中发现了`xlsx`、`pdf`、`docs`等，跟 `Claude Skills` 官方的技能库很像：

https://github.com/anthropics/skills/tree/main/skills

我们可以继续让 glm 帮我们分析下到底有没有关系。

> 提示词：
> 帮我分析下目录：/skills，里面的技能包，有多少个技能是跟.claude/skills这个目录一样的，要确保里面的 skills.md 要一模一样。

![](https://mmbiz.qpic.cn/mmbiz_png/6Q6lA5HbrOwowxsJJnpnibm6GxWnPjnE0E72RC9cyZh2VnZdo0iaD2khH5DHjWib6Vb0Vjicj65c2VwP9nV2aibxR8Q/640?wx_fmt=png&from=appmsg)

很明显得出一个结论，`Z.ai`里面的 14 个技能包，有 5 个跟官方一样，有 1 个`frontend-design`是要优于 Claude Skills 的版本的，`Z.ai`做了有详细的优化。

我们继续分析剩余的 **8** 个技能包：

```
### AI & 媒体处理技能 (6个)
1. **ASR** - 语音识别 (自动语音转文字)
   - SKILL.md (技能说明文档)
   - scripts/asr.ts (示例脚本)

2. **LLM** - 大语言模型 (智能对话、文本生成)
   - SKILL.md (技能说明文档)
   - scripts/chat.ts (示例脚本)

3. **TTS** - 文字转语音
   - SKILL.md (技能说明文档)
   - tts.ts (示例脚本)

4. **VLM** - 视觉语言模型 (图像理解)
   - SKILL.md (技能说明文档)
   - scripts/vlm.ts (示例脚本)

5. **image-generation** - 图像生成
   - SKILL.md (技能说明文档)
   - scripts/image-generation.ts (示例脚本)

6. **video-generation** - 视频生成
   - SKILL.md (技能说明文档)
   - scripts/video.ts (示例脚本)

### Web & 信息检索技能 (2个)
7. **web-reader** - 网页内容读取
   - SKILL.md (技能说明文档)
   - scripts/web-reader.ts (示例脚本)

8. **web-search** - 网页搜索
   - SKILL.md (技能说明文档)
   - scripts/web_search.ts (示例脚本)
```

我们看到这 8 个技能包，都要相应的`示例脚本`。

关于脚本的代码示例，我发现了一样东西：`import ZAI from 'z-ai-web-dev-sdk';`

```
import ZAI from 'z-ai-web-dev-sdk';

interface PageReaderFunctionResult {
  code: number;
  data: {
    html: string;
    publishedTime?: string;
    title: string;
    url: string;
    usage: {
      tokens: number;
    };
  };
  meta: {
    usage: {
      tokens: number;
    };
  };
  status: number;
}
```

通过 ai 分析，我们可以知道，想要使用上面 8 个技能包，你要用对应的 `ts SDK`。

![](https://mmbiz.qpic.cn/mmbiz_png/6Q6lA5HbrOwowxsJJnpnibm6GxWnPjnE00HwnicutficMiaY8nrsQ3iadasM3upGptrnibhzEpOPheWL28Ps5ML5k7HA/640?wx_fmt=png&from=appmsg)

分析到这里我大概知道了，这 8 个技能包，都是给`Z.ai`官网上面的全栈开发的工作环境（虚拟服务器环境）使用的，不适合我们本地环境使用。

## 写到最后

回顾这段“邪修”之路，其实就是一个打破思维定式的过程：

### 我的邪修之路总结

#### 1. 起点：从官方宣传的困惑开始

*   遇到问题：官方铺天盖地的宣传，却找不到实际可用的Skills资源
*   常规思路：GitHub、搜索引擎、爬虫...都想了一遍
*   **突破点**：既然找不到，那就直接问AI本身

#### 2. 转折：一句提示词的魔力

*   最简单的方式往往最有效：`“你有什么技能”`
*   三个步骤搞定：获取列表 → 汇总打包 → 直接下载
*   **省去了**：漫无边际的搜索、复杂的爬虫代码、翻遍文档的痛苦

#### 3. 发现：14个技能包的真相

*   5个来自Claude Skills官方库（直接复用）
*   1个是优化版本（frontend-design）
*   8个专属Z.ai的AI媒体处理技能（需要特定SDK环境）

### 💡 邪修的启示

> **有时候，最直接的问题，能得到最直接的答案。**

我们习惯了：

*   ❌ 去GitHub搜仓库
*   ❌ 去搜索引擎翻文档
*   ❌ 写爬虫抓数据

但其实：

*   ✅ 直接问AI：`“你有什么技能”`
*   ✅ 直接让AI打包：`“把所有文件汇总给我”`
*   ✅ 直接下载使用

**这就是邪修的精髓**：不走寻常路，用最简单的方式解决最复杂的问题。
