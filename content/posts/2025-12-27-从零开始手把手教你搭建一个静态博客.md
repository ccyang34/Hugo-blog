## 前言

你是否曾想过拥有一个属于自己的博客，记录技术心得、生活感悟，却苦于不懂复杂的后端开发？

静态博客生成器或许是你最好的选择。它无需数据库，部署简单，速度快，并且完全免费。今天，我们就来手把手教你，使用 **Hugo** 和 **GitHub Pages**，从零开始搭建一个高性能的静态博客。

## 准备工作

在开始之前，你需要准备以下几样东西：

1.  **一个 GitHub 账号**：用于存放你的博客源代码和托管网站。
2.  **安装 Git**：用于版本控制和代码推送。
3.  **安装 Hugo**：我们选择的静态博客生成器。

### 安装 Hugo

Hugo 的安装非常简单，以 macOS 为例，使用 Homebrew 一行命令即可搞定：

```bash
brew install hugo
```

安装完成后，在终端输入 `hugo version`，如果显示版本号，说明安装成功。

Windows 和 Linux 用户请参考 [Hugo 官方安装指南](https://gohugo.io/getting-started/installing/)。

## 第一步：创建博客项目

打开终端，执行以下命令来创建一个新的 Hugo 站点，我们将其命名为 `myblog`：

```bash
hugo new site myblog
cd myblog
```

这会在当前目录下生成一个名为 `myblog` 的文件夹，里面包含了 Hugo 站点的基本结构。

## 第二步：选择并安装主题

Hugo 有丰富的主题库。我们以一个简洁优雅的主题为例。进入你的博客目录，初始化 Git 仓库，并将主题添加为子模块：

```bash
git init
git submodule add https://github.com/theNewDynamic/gohugo-theme-ananke.git themes/ananke
```

然后，编辑站点配置文件 `config.toml`，指定使用的主题：

```toml
baseURL = "https://yourusername.github.io/" # 稍后替换为你的 GitHub 用户名
languageCode = "zh-cn"
title = "我的技术博客"
theme = "ananke"
```

## 第三步：创建你的第一篇文章

使用 Hugo 命令快速创建一篇博文：

```bash
hugo new posts/my-first-post.md
```

这条命令会在 `content/posts/` 目录下生成一个 Markdown 文件。用你喜欢的文本编辑器打开它，开始写作吧！

Hugo 文章采用 **Front Matter** + **Markdown 正文** 的格式。Front Matter 是文章元数据，用 `---` 包裹。

```yaml
---
title: "我的第一篇文章"
date: 2023-10-27T11:30:00+08:00
draft: false # 设为 false 表示发布文章
---

这里是文章的正文，使用 **Markdown** 语法书写非常方便。

- 列表项一
- 列表项二

> 这是一段引用。

![这是一张图片](https://example.com/image.jpg)
```

## 第四步：本地预览

在博客根目录下运行 Hugo 本地服务器：

```bash
hugo server -D
```

`-D` 参数表示同时渲染草稿（`draft: true`）的文章。打开浏览器，访问 `http://localhost:1313`，你就能看到博客的实时预览效果了！边写边看，体验极佳。

## 第五步：部署到 GitHub Pages

这是最关键的一步，让你的博客被全世界看到。

### 1. 创建 GitHub 仓库

在 GitHub 上创建一个新的公共仓库，命名为 `yourusername.github.io`（请将 `yourusername` 替换为你的 GitHub 用户名）。

### 2. 生成静态文件并推送

首先，确保你的 `config.toml` 中的 `baseURL` 已经正确设置为 `https://yourusername.github.io/`。

然后，在博客根目录执行以下命令生成最终的静态网站文件：

```bash
# 停止本地服务器（Ctrl+C），然后运行：
hugo
```

这会在项目根目录下生成一个 `public` 文件夹，里面就是所有可以被直接访问的 HTML、CSS 等文件。

接下来，我们将 `public` 目录的内容推送到 GitHub 仓库的主分支。一个常见的做法是，将 `public` 目录设置为一个独立的 Git 子模块，或者使用 GitHub Actions 自动化部署。这里介绍手动推送的方法：

```bash
cd public
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/yourusername/yourusername.github.io.git
git push -u origin main
```

### 3. 访问你的博客

推送完成后，稍等片刻（通常不超过2分钟），访问 `https://yourusername.github.io`，你的个人博客就正式上线了！

## 进阶与优化

- **自定义域名**：在仓库设置中绑定你自己的域名。
- **使用 GitHub Actions 自动化**：每次推送文章源码到 `main` 分支，自动构建并部署 `public` 文件夹，无需手动操作。
- **探索更多主题和配置**：Hugo 官方主题库有数百款主题等你挑选。
- **添加评论系统**：可以集成 Gitalk、Utterances 等基于 GitHub 的评论工具。

## 结语

恭喜你！你已经成功搭建了一个快速、安全、完全属于自己的静态博客。

这套组合（Hugo + GitHub Pages）的优势在于：

- **完全免费**：GitHub Pages 提供免费的托管和 HTTPS 证书。
- **极速访问**：静态文件被 CDN 加速，全球访问速度都很快。
- **专注写作**：只需用 Markdown 写好文章，剩下的交给 Hugo 和 Git。
- **高度可控**：所有数据（文章、配置）都掌握在自己手中，没有平台限制。

从此，你可以尽情地书写和分享。开始你的博客之旅吧！
