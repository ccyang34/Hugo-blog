# 博客编辑与生成规范指南

为了保持博客内容的整洁和脚本的正常运行，请遵循以下文件存放和引用规范。

## 1. 文件存放位置

| 类型 | 存放路径 | 说明 |
| :--- | :--- | :--- |
| **Markdown 文章** | `content/posts/` | 所有博客内容文件（.md）均放于此。 |
| **常规图片/附件** | `static/img/` | 博客中使用的常规图片。 |
| **分析图表** | `static/img/charts/` | 脚本自动生成的图表文件（.png, .jpg）。 |

## 2. Markdown 文章规范 (Front Matter)

每篇文章开头必须包含 YAML 格式的配置信息，示例如下：

```yaml
---
title: "文章标题"
date: 2024-03-27 18:00
image: img/charts/example.png  # 封面图路径，相对于 static/ 目录
description: "简短的文章描述"
categories:
    - 市场分析                   # 分类项目
tags:
    - 豆油                       # 标签项目
---
```

## 3. 图片引用方式

在 Markdown 正文中引用图片时，路径需从 `static/` 之后的目录开始写起，并以 `/` 开头：

- **引用图表**: `![描述](/img/charts/your_chart.png)`
- **引用常规图片**: `![描述](/img/your_image.jpg)`

## 4. 自动化脚本使用

目前项目中包含以下快捷脚本：

- **生成分析博客**: 运行 `python3 分析文章脚本/自动生成榨利博客.py`。
  - 它会自动获取数据、保存图片到 `static/img/charts/` 并创建文章。
- **一键同步博客**: 运行 `./sync_blog.sh` 或 `python3 sync_blog.py`。
  - 它会自动执行 `Git add`, `Commit` 和 `Push`。

## 5. 建议工作流

1. **手动写稿**: 在 `content/posts/` 新建 .md -> 粘贴模板 -> 编写正文。
2. **预览**: 本地运行 `hugo server` 查看效果。
3. **发布**: 运行 `./sync_blog.sh` 同步到远程服务器。
